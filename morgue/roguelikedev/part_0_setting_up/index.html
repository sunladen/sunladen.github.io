roguelike.html

roguelike.html defines a title, style and pulls in the games bundled JavaScript, roguelike.js

<html>
  <head>
    <title>The complete roguelike tutorial</title>
    <style>
      body { padding: 0; margin: 0 }
      * { box-sizing: border-box; }
    </style>
  </head>
  <body>
    <script src="roguelike.js"></script>
  </body>
</html>

roguelike.js

Source code is organised into modules that get bundled into a single browser friendly file. I'm using the module bundler Rollup to do this.

rollup.config.js

My Rollup configuration sources main.js as an entry point module and adds a sourceMap to the bundle output to help debug the game in the browser.

export default {
    entry: 'src/main.js',
    indent: '\t',
    sourceMap: 'inline',
    dest: 'roguelike.js',
    format: 'iife'
}

main.js

main.js at this point in the tutorial could probably simply be an empty file, but lets import rot.js, create a ROT.Display and add its container to the page.

import { ROT } from "./rot"

let display = new ROT.Display()
document.body.appendChild( display.getContainer() )
